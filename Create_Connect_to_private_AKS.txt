Connect to Azure VM from Filezilla:


Steps to Connect via SFTP (Recommended)
Get VM Details: In the Azure Portal, locate your Virtual Machine to find its Public IP address.
Open FileZilla: Open the FileZilla Client, go to File > Site Manager .
Configure Site:
Protocol: Select SFTP - SSH File Transfer Protocol.
Host: Enter your Azure VM public IP address.
Port: Enter 22(default SSH port).
Logon Type: Select Key file.
User: Enter your VM username.
Key file: Browse and select the .pemor .ppkprivate key file used to create the VM.
Connect: Click Connect . If asked to trust the host key, check "Always trust" and click OK.

Create Argo Rollouts:

kubectl create namespace argo-rollouts
kubectl apply -n argo-rollouts -f https://github.com/argoproj/argo-rollouts/releases/latest/download/install.yaml


Create ArgoCD Image Updater:

kubectl create namespace argocd-shadow

This is ARGOCD updater that takes directly from github chnage after push



kubectl apply -n argocd-shadow -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

kubectl apply --server-side --force-conflicts -n argocd-shadow -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml


This is ARGOCD Image Updater that picks up chnages after the image in pushed to docker repo:

kubectl apply -n argocd-shadow -f https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/config/install.yaml

Stable version of Image Updaterthat does not need CRDs:

kubectl apply -n argocd-shadow -f https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/v0.12.2/manifests/install.yaml



kubectl get pods -n argocd-shadow

Create Image Updater Secrets:

kubectl create secret generic acr-registry-secret -n argocd-shadow \
  --from-literal=token=<ACR_USERNAME>:<ACR_PASSWORD>

Apply the Image Updater:

kubectl delete -f application.yaml
kubectl apply -f application.yaml

Verify If the application is running:

kubectl describe application azure-rag-app-shadow -n argocd-shadow

Watch the Rollout:

kubectl argo rollouts get rollout azure-rag-app-shadow --watch








In Case doing from private github:

# 1. Create the secret with your credentials

# Replace <GITHUB_USERNAME> and <GITHUB_TOKEN> with your actual details

kubectl delete secret repo-creds -n argocd

kubectl create secret generic repo-creds -n argocd \
  --from-literal=url=https://github.com/kaustav4284/AZURE_RAG_APP_AKS.git \
  --from-literal=password=<GITHUB_TOKEN> \
  --from-literal=username=<GITHUB_USERNAME>


# 2. Label it so Argo CD knows it's for repository access

kubectl label secret repo-creds -n argocd "argocd.argoproj.io/secret-type=repository"



Install Istioctl:

curl -L https://istio.io/downloadIstio | sh -
# 2. Extract and move to the Istio directory (it will be created by the sh script)
# The version might vary, for example: cd istio-1.21.0
cd istio-*
# 3. Add istioctl to your path

# 4. Verify installation
istioctl version --remote=false

istioctl install
